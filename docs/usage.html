Usage Guide

Learn how to use Waqt for time tracking

Web Interface

Navigate to http://localhost:5555 after running waqt

  • Dashboard Timer: One-click start/stop/pause/resume
  • Calendar Overview: Monthly view with color-coded days for work entries, leave, and missing entries
  • Time Entry Management: Add, view, and delete entries
  • Reports: Weekly and monthly summaries
  • Leave Management: Track vacation and sick days with multi-day selection

CLI Commands

# Start tracking time
waqt start --time 09:00 --description "Working on feature X"

# End tracking time
waqt end --time 17:30

# Add a completed time entry (automatically deducts default 45min pause)
waqt add --start 09:00 --end 17:00 --description "Development work"

# Add entry with no pause
waqt add --start 09:00 --end 17:00 --pause none

# Add entry with custom pause (e.g., 30 minutes)
waqt add --start 09:00 --end 17:00 --pause 30

# View summary
waqt summary

# Export time entries (default: CSV)
waqt export --period week

# Export to JSON
waqt export --period month --format json

# Export to Excel
waqt export --period all --format excel --output backup.xlsx

# Request multi-day leave (excludes weekends)
waqt leave-request --from 2026-01-13 --to 2026-01-19

# Configuration management
waqt config list
waqt config set weekly_hours 40

Multi-Day Leave Management

Request leave spanning multiple days with automatic weekend exclusion and working hours calculation.

Web Interface

  • Date Range Selection: Choose start and end dates for your leave
  • Live Preview: See total days, working days, and excluded weekends in real-time
  • Automatic Calculation: Working hours calculated based on your configured hours per day
  • Weekend Exclusion: Saturdays and Sundays are automatically excluded from leave records

Command Line

# Request vacation for a full week (Mon-Sun → creates 5 records for Mon-Fri)
waqt leave-request --from 2026-01-13 --to 2026-01-19 --type vacation --desc "Winter break"

# Request sick leave with description
waqt leave-request --from 2026-01-20 --to 2026-01-24 --type sick --desc "Medical leave"

# Single day leave (backward compatible)
waqt leave-request --from 2026-01-15 --to 2026-01-15

How It Works

  • Working Days Only: Only Monday-Friday are counted as working days
  • Weekend Detection: Saturdays and Sundays are automatically detected and excluded
  • Clear Feedback: See exactly which days will be recorded before confirming
  • Example: Jan 12-19 (Mon-Sun) = 6 working days, 2 weekends excluded, 48 hours

Category Management

Organize your time entries with custom categories for projects, clients, or activity types.

Web Interface

  • Manage Categories: Create, edit, and delete categories via the new "Categories" page.
  • Assign Categories: Select a category when adding or editing a time entry.
  • Reporting: View a breakdown of hours by category in the Reports section.

Command Line (Coming Soon)

CLI support for category management is planned for a future update.

Features

  • Custom Colors: Assign colors to categories for visual distinction in reports.
  • Codes: Use short codes for quick reference.
  • Validation: Prevents deletion of categories currently in use.

Data Export

Export your time tracking data in multiple formats for reporting, analysis, or backup.

Web Interface

  • Formats: Export to CSV, JSON, or Excel directly from the Reports page.
  • Context-Aware: Exports data for the currently selected week or month view.
  • Excel Features: Excel exports include formatted sheets with auto-adjusted columns and summary statistics.

Command Line

# Standard CSV export
waqt export --period week --output report.csv

# JSON export for processing
waqt export --period month --format json --output data.json

# Excel export for sharing
waqt export --period all --format excel --output full_history.xlsx
