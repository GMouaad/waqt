Usage Guide

Learn how to use Waqt for time tracking

Web Interface

Navigate to http://localhost:5555 after running waqt

  • Dashboard Timer: One-click start/stop/pause/resume
  • Time Entry Management: Add, view, and delete entries
  • Reports: Weekly and monthly summaries
  • Leave Management: Track vacation and sick days with multi-day selection

CLI Commands

# Start tracking time
waqt start --time 09:00 --description "Working on feature X"

# End tracking time
waqt end --time 17:30

# View summary
waqt summary

# Export time entries
waqt export --period week

# Request multi-day leave (excludes weekends)
waqt leave-request --from 2026-01-13 --to 2026-01-19

# Configuration management
waqt config list
waqt config set weekly_hours 40

Multi-Day Leave Management

Request leave spanning multiple days with automatic weekend exclusion and working hours calculation.

Web Interface

  • Date Range Selection: Choose start and end dates for your leave
  • Live Preview: See total days, working days, and excluded weekends in real-time
  • Automatic Calculation: Working hours calculated based on your configured hours per day
  • Weekend Exclusion: Saturdays and Sundays are automatically excluded from leave records

Command Line

# Request vacation for a full week (Mon-Sun → creates 5 records for Mon-Fri)
waqt leave-request --from 2026-01-13 --to 2026-01-19 --type vacation --desc "Winter break"

# Request sick leave with description
waqt leave-request --from 2026-01-20 --to 2026-01-24 --type sick --desc "Medical leave"

# Single day leave (backward compatible)
waqt leave-request --from 2026-01-15 --to 2026-01-15

How It Works

  • Working Days Only: Only Monday-Friday are counted as working days
  • Weekend Detection: Saturdays and Sundays are automatically detected and excluded
  • Clear Feedback: See exactly which days will be recorded before confirming
  • Example: Jan 12-19 (Mon-Sun) = 6 working days, 2 weekends excluded, 48 hours
